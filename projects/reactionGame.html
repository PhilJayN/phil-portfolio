How the Game Works

At the core of the game's code is a .render() method whose primary job is, just like its name suggests, is to render the target for the user to hit.

Here is a tiny, condensed code snippet:

{% highlight ruby %} var App = { render: function() { var posX = (Math.random() * (wMax - wBox)).toFixed(); var posY = (Math.random() * (hMax - hBox)).toFixed(); $bullseye.css({ 'position': 'absolute', 'left': posX + 'px', 'top': posY + 'px', }); }, } {% endhighlight %}

The variables posX and posY will have a new random number assigned when the .render() method is called. The properties 'left' and 'top' gets assigned whatever number is generated from the two aforementioned variables. Then the target will be rendered in its new position!

{% highlight ruby %} $bullseye.css({ 'position': 'absolute', 'left': posX +'px', 'top': posY + 'px', }); {% endhighlight %}

Challenges

During the development process, one of the challenge is keeping the target inside its container. Because the target's position is set to absolute in CSS, it does not stay within the confiment of its parent div. To solve this problem, the following steps were used:

  • Find the size of the game container(ex: 200px) and subtract the target size(ex: 50px) from it, resulting in 150px
  • Generate a "random" number using Math.random() and multiply it by 150px from above step

That's it!
