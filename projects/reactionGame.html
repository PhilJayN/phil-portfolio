Code Snippet

At the core of the game's code is a .render() method whose primary job is to render the target for the user to hit. 


var App = {

  render: function() {

    var $bullseye = $('.bullseye');
    var $gameBox = $('.game-box');

    var gameBoxW = $gameBox.innerWidth();
    var gameBoxH = $gameBox.innerHeight();
    var bullseyeW = $bullseye.innerWidth();
    var bullseyeH = $bullseye.innerHeight();

    var posX = (Math.random() * (gameBoxW - bullseyeW)).toFixed();
    var posY = (Math.random() * (gameBoxH - bullseyeH)).toFixed();

    $bullseye.css({
      'position': 'absolute',
      'left': posX + 'px',
      'top': posY + 'px',
    });
  },

}

The variables posX and posY will have a new random number assigned when the .render() method is called. The properties 'left' and 'top' gets assigned the value from the aforementioned variables. Then the target will be rendered in its new position!

Challenges

During the development process, one of the challenge is keeping the target inside its container. Because the target's position is set to absolute in CSS, it does not stay within the confiment of its parent div. To solve this problem, the following steps were used:

  • Find the size of the game container(ex: 200px) and subtract the target size(ex: 50px) from it, resulting in 150px
  • Generate a "random" number using Math.random() and multiply it by 150px from above step

That's it!
