Global Variables and Its Complications

I wish I knew about some of the bad practices with writing JavaScript before actually creating my projects. Currently, my gas calculator has many global variables. I'm in the process of rewriting some of the code to fix this issue. As stated in Douglas Crockford's book, JavaScript The Good Parts, global variables can be convienient in small programs, but they can become very unweildy in larger programs. Global variables can be changed by other parts of the program, so this can pose a problem. You probably have the same question as I do: "If global variables are so bad, what are the alternatives then?"

Currently, I do not have the answer, but I'm researching more, so stay tuned.
